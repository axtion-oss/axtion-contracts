# axtion-contracts

AI-generated smart contract templates deployed by the Axtion pipeline. All contracts target Base mainnet and are generated by Claude based on tweet commands.

## Templates

| Contract | File | Description |
|---|---|---|
| ERC-20 Token | `src/token/ERC20Token.sol` | Mintable/burnable fungible token |
| NFT Collection | `src/nft/ERC721Collection.sol` | ERC-721 with URI storage and mint price |
| Staking Pool | `src/staking/StakingPool.sol` | Single-token staking with configurable APR |
| Governance DAO | `src/dao/GovernanceDAO.sol` | Token-weighted proposal and voting |

## Requirements

- [Foundry](https://getfoundry.sh)
- OpenZeppelin Contracts v5

## Setup

```bash
git clone https://github.com/axtion-oss/axtion-contracts
cd axtion-contracts
forge install OpenZeppelin/openzeppelin-contracts@v5.0.0
cp .env.example .env
# fill in PRIVATE_KEY and BASE_RPC_URL
```

## Test

```bash
forge test
```

## Deploy

```bash
forge script script/Deploy.s.sol --rpc-url $BASE_RPC_URL --private-key $PRIVATE_KEY --broadcast
```

## Verify on Basescan

```bash
forge verify-contract <address> src/token/ERC20Token.sol:ERC20Token \
  --chain base \
  --etherscan-api-key $BASESCAN_API_KEY
```

## How it works

When you tweet a command to @axtion, the pipeline:

1. Sends your command to Claude for contract generation
2. Creates a GitHub repo under your org with the generated Solidity + Foundry tests
3. Compiles with solc and deploys to Base mainnet
4. Launches a project token

These templates are the canonical starting points Claude uses. Real builds may vary based on the command.
